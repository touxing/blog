阅读源码心得
- 读源码，先从整体入手，不必一开始拘泥于细节。
- 整体阅读下来，了解主要流程，每一模块是干什么的即可，需要的时候再回台看实现
- 读细节的时候，写代码注释，思考每个代码块的功能，理解作者为什么这样写，哪些是可以借鉴的
> 阅读源码 不是为了把源码写出来，是为了学习思想，实现是次要的，思想是主要的
